# 한국어 CS 문의 라벨링 가이드 v1

## 1. 목적과 범위

이 가이드는 한 개의 한국어 고객 문의를 7개 중 정확히 하나의 라벨로 분류하기 위한 기준이다.
모델은 상담 답변, 환불 승인, 고객 자동 배정, 개인정보 판단을 하지 않는다. 입력 문장에 드러난
**주된 문의 의도**만 분류한다.

## 2. 공통 판정 규칙

1. 한 문의에는 라벨을 하나만 부여한다.
2. 복합 문의는 고객이 먼저 해결받기를 원하는 문제 또는 후속 조치를 먼저 결정하는 문제를 고른다.
3. 상품 하자 때문에 반품하려는 경우는 `exchange_return`이다. 결제 후 주문을 취소해 달라는 경우는
   `cancellation_refund`이다.
4. 앱이나 웹 화면의 비정상 동작이 원인인 경우는 문제의 대상이 결제·배송 자체가 아니라면
   `technical_issue`를 우선한다.
5. 주된 의도가 문장만으로 정해지지 않는 동등한 복합 문의는 학습용 데이터에서 제외하고
   `boundary_review`로 기록한다.
6. 사람 이름, 전화번호, 이메일, 주문번호 등 개인정보는 실제 값 대신 마스킹 또는 가상값만 사용한다.

## 3. 라벨 정의

### payment — 결제

- 포함: 결제 실패, 결제 승인·중복 청구, 결제수단, 결제 승인 뒤 주문 미생성.
- 제외: 주문 또는 결제 취소·환불 요청(`cancellation_refund`), 결제 화면 자체의 오류가 핵심인 경우
  (`technical_issue`).
- 대표 예시:
  1. 카드 결제가 계속 실패합니다.
  2. 결제는 됐는데 주문 내역이 없어요.
  3. 같은 주문 금액이 두 번 청구됐습니다.
  4. 간편결제 수단을 등록할 수 없어요.
  5. 결제 승인 문자는 왔는데 주문이 생성되지 않았습니다.
  6. 무통장입금 확인은 언제 되나요?
  7. 할부 개월 수를 바꾸고 싶습니다.
  8. 결제 금액이 주문서 금액과 다릅니다.
  9. 해외 카드로 결제할 수 있나요?
  10. 결제 취소 문자는 받았는데 카드 승인만 남아 있어요.

### cancellation_refund — 취소·환불

- 포함: 주문 취소, 결제 취소, 환불 신청·진행 상태·지연·금액 문의.
- 제외: 수령한 상품을 반품하는 절차(`exchange_return`), 아직 결제되지 않은 일반 주문 변경 문의.
- 대표 예시:
  1. 주문을 취소하고 싶어요.
  2. 환불은 언제 입금되나요?
  3. 이미 결제한 주문을 취소할 수 있나요?
  4. 부분 취소가 가능한지 알려 주세요.
  5. 환불 금액이 예상보다 적습니다.
  6. 취소했는데 카드 결제가 아직 남아 있습니다.
  7. 배송 전에 주문을 취소해 주세요.
  8. 환불 신청 상태가 계속 처리 중입니다.
  9. 쿠폰을 쓴 주문도 취소할 수 있나요?
  10. 결제 취소 완료 문자를 받지 못했습니다.

### delivery — 배송

- 포함: 출고 여부, 송장 조회, 배송 지연, 분실, 오배송, 배송지 변경, 수령 관련 문의.
- 제외: 수령 후 교환·반품 요청(`exchange_return`), 주문 생성 또는 결제 문제(`payment`).
- 대표 예시:
  1. 주문한 상품이 아직 출고되지 않았어요.
  2. 송장 번호로 조회가 되지 않습니다.
  3. 배송 예정일보다 늦게 오고 있어요.
  4. 배송지를 변경하고 싶습니다.
  5. 상품이 다른 주소로 배송됐습니다.
  6. 택배가 분실된 것 같아요.
  7. 부재중이라 수령 방법을 바꾸고 싶습니다.
  8. 제주도도 배송 가능한가요?
  9. 여러 상품이 나눠서 배송되나요?
  10. 배송 완료라고 나오는데 물건을 못 받았습니다.

### exchange_return — 교환·반품

- 포함: 수령 상품의 교환·반품 신청, 가능 기간, 회수, 비용, 진행 상태, 상품 하자.
- 제외: 배송 전 주문 취소(`cancellation_refund`), 배송 중 위치·지연(`delivery`).
- 대표 예시:
  1. 받은 상품의 색상을 교환하고 싶어요.
  2. 사이즈가 맞지 않아 반품하려고 합니다.
  3. 상품이 파손되어 도착했습니다.
  4. 반품 수거는 언제 오나요?
  5. 교환 상품은 언제 다시 보내 주나요?
  6. 반품 배송비는 누가 부담하나요?
  7. 포장을 버렸는데 반품할 수 있나요?
  8. 교환 신청 상태가 멈춰 있습니다.
  9. 다른 상품이 와서 교환 요청합니다.
  10. 수령 후 며칠까지 반품할 수 있나요?

### account — 계정

- 포함: 가입, 로그인, 비밀번호, 본인 인증, 회원 정보 수정, 탈퇴, 계정 잠금.
- 제외: 앱 전체 오류(`technical_issue`), 주문·배송·결제 상태 조회.
- 대표 예시:
  1. 비밀번호를 잊어버렸습니다.
  2. 로그인이 계속 실패해요.
  3. 휴대폰 번호를 변경하고 싶습니다.
  4. 회원 탈퇴는 어떻게 하나요?
  5. 본인 인증 문자가 오지 않습니다.
  6. 계정이 잠긴 것 같아요.
  7. 이메일 주소를 수정하고 싶습니다.
  8. 가입한 적 없는데 계정이 있다고 나옵니다.
  9. 소셜 로그인 연결을 해제하고 싶어요.
  10. 내 계정의 로그인 기록을 확인할 수 있나요?

### technical_issue — 기술 오류

- 포함: 앱·웹의 오류, 화면 멈춤, 기능 미작동, 알 수 없는 오류 메시지, 쿠폰·검색·장바구니 등 기능 오류.
- 제외: 결제 승인·청구·주문 미생성(`payment`), 계정 자격 증명·회원 정보 문제(`account`).
- 대표 예시:
  1. 앱을 열면 바로 종료됩니다.
  2. 장바구니에 상품이 담기지 않아요.
  3. 주문 페이지가 계속 로딩 중입니다.
  4. 쿠폰 적용 버튼을 눌러도 반응이 없습니다.
  5. 검색 결과가 화면에 표시되지 않습니다.
  6. 오류 코드 E500이 계속 나옵니다.
  7. 앱 업데이트 후 주문 내역 화면이 비어 있어요.
  8. 웹사이트에서 이미지가 전혀 보이지 않습니다.
  9. 알림 설정을 저장할 수 없습니다.
  10. 특정 버튼을 누르면 흰 화면만 나옵니다.

### product_general — 상품·일반 문의

- 포함: 상품 상세 정보, 옵션, 호환성, 재고, 가격, 할인·프로모션 정책, 일반 이용 안내.
- 제외: 상품 수령 후 교환·반품(`exchange_return`), 쿠폰 버튼·화면의 비정상 동작(`technical_issue`).
- 대표 예시:
  1. 이 상품의 정확한 색상은 무엇인가요?
  2. 다음 주에 재입고되나요?
  3. 이 제품은 어떤 기종과 호환되나요?
  4. 선물 포장이 가능한가요?
  5. 상품 구성품을 알려 주세요.
  6. 할인 행사는 언제까지 하나요?
  7. 두 개를 사면 추가 할인이 있나요?
  8. 옵션별 재고를 확인하고 싶습니다.
  9. 상품 보증 기간이 있나요?
  10. 정기구독 상품의 이용 방법을 알려 주세요.

## 4. 경계 사례 20개

| 문의 | 라벨 | 판정 이유 |
|---|---|---|
| 결제는 됐는데 주문이 없어요. | payment | 승인 후 주문 미생성이 핵심이다. |
| 배송 전에 주문을 취소하고 싶어요. | cancellation_refund | 배송 상태가 아니라 취소 요청이 핵심이다. |
| 배송된 상품이 깨져서 환불받고 싶어요. | exchange_return | 수령 상품 하자로 인한 반품 절차가 우선이다. |
| 카드 청구가 두 번 됐으니 한 건을 돌려주세요. | payment | 중복 청구 문제를 먼저 해결해야 한다. |
| 앱에서 결제 버튼을 누르면 흰 화면입니다. | technical_issue | 결제 승인 실패가 아니라 화면 기능 오류다. |
| 로그인 후 주문 내역이 보이지 않아요. | technical_issue | 계정 인증보다 주문 내역 화면의 비정상 동작이 핵심이다. |
| 비밀번호를 바꾸려는데 인증 문자가 안 와요. | account | 계정 복구 절차의 일부다. |
| 쿠폰이 적용되지 않아 결제 금액이 달라요. | technical_issue | 쿠폰 적용 기능 미작동이 명시돼 있다. |
| 쿠폰 할인 정책이 궁금합니다. | product_general | 기능 오류가 아닌 정책 문의다. |
| 주문한 색상과 다른 상품이 왔어요. | exchange_return | 수령 상품 교환의 문제다. |
| 배송 완료인데 물건이 없어요. | delivery | 배송·수령 확인 이슈다. |
| 반품했는데 환불이 안 됐어요. | cancellation_refund | 반품 절차는 완료됐고 환불 상태가 핵심이다. |
| 출고 전인데 주소를 바꿀 수 있나요? | delivery | 배송지 변경 요청이다. |
| 상품이 품절인데 결제가 되었어요. | payment | 결제 처리·주문 상태를 먼저 확인해야 한다. |
| 계정을 탈퇴하면 환불은 어떻게 되나요? | boundary_review | 계정·환불 중 어느 조치가 주된지 문장만으로 불명확하다. |
| 배송도 안 오고 취소도 안 됩니다. | boundary_review | 동등한 복합 이슈라 주된 의도가 없다. |
| 앱 오류 때문에 주문을 취소하고 싶어요. | cancellation_refund | 명시적 고객 요청은 주문 취소다. |
| 상품 상세 페이지가 열리지 않습니다. | technical_issue | 상품 정보가 아니라 페이지 기능 오류다. |
| 반품 배송비 결제 방법을 알려 주세요. | exchange_return | 반품 절차에 부수되는 비용 문의다. |
| 환불된 금액이 카드 명세서에 안 보여요. | cancellation_refund | 환불 반영 상태 문의다. |

## 5. 데이터 작성·검수 체크리스트

- [ ] 문장만 보고 라벨을 하나로 판단할 수 있는가?
- [ ] 같은 라벨 문장이 특정 명사나 고정 문구만 반복하지 않는가?
- [ ] 실제 개인정보나 실제 주문 식별자가 없는가?
- [ ] `source`, `is_synthetic`, `review_status`, `notes`가 채워졌는가?
- [ ] Train/Validation/Test 간에 같은 템플릿군이 섞이지 않도록 원본 관계를 기록했는가?
- [ ] 경계 사례는 일반 학습 데이터와 구분해 별도 검수했는가?
