## Confidence 해석

가장 높은 오답 confidence도 0.195로, 0.7 이상처럼 높은 확신의 오답은 0건이다. 이는 모델이
이 사례들에서 확실히 잘못된 규칙을 배웠다기보다, 7개 클래스 중 어느 것도 강하게 선택하지 못한
상태에 가깝다. 다만 낮은 confidence만으로 안전하다고 판단할 수는 없으므로 운영 단계에서는
신뢰도 임계값과 사람 검토 경로가 필요하다.

## 개선 우선순위

1. `cancellation_refund` 데이터를 우선 확대한다. 출고 전 취소, 카드 승인 취소, 환불 지연,
   부분 취소, 반품 완료 후 환불처럼 결제·배송과 겹치는 표현을 다양하게 추가한다.
2. `delivery`와 `exchange_return`의 역배송·회수·수령 지시 표현을 보강한다.
3. `product_general`의 쿠폰 정책·가격·재고·상품 정보 문장을 전용 라벨 기준에 맞게 추가한다.
4. 새 데이터를 추가하면 기존 Test는 바꾸지 않는다. 별도의 v2 Train/Validation/Test를 만들고
   라벨 가이드 변경 여부부터 검토한다.

## 결론

이번 오류는 주로 데이터 부족과 경계 표현의 부족으로 설명되며, 현재 라벨 가이드 자체를 즉시
바꿔야 한다는 증거는 없다. 우선순위는 모델 크기 증가가 아니라 경계 사례를 포함한 데이터 품질
개선이다.
